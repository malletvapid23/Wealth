CMRP Amount: The Fountain

CMRP Rate determines the size of the pool in our application. It works like value added tax; when the number of sales gets bigger, CMRP Amount, the amount that we can control through the pool gets bigger as well.

At this stage, our application will be used as a simulation tool; there will be more and more samples with various cases to show how we could improve the system. Within these simulations, we can safely experiment with the sizes of the pools, by making them too big or too small.

Since the development process will take a while, the details of possible implementations in real life can be discussed later.
However, to give some quick ideas;

  1. If the results will become reliable, for a certain CMRP, the money can be collected from the volunteer users and can be distributed to the organizations within that pool in a certain period time.
  2. If any government can be convinced to use this system, initially they can allocate a certain amount and the system can be used by the citizens of that country. Ultimately and ideally, this system should be implemented or at least aligned with the tax system of each country, as in our application.
None of the options is easy and can be achieved in a short period of time. Therefore, the first step is to focus on creating a solid application with reliable ratings. If this approach can be proved to be worthy in time, we can assume that the rest will follow.
